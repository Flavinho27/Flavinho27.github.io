Introduction Cette veille technologique me permet d’avoir les connaissances théoriques nécessaires quant à l’utilisation des différentes technologies utilisées durant les 2 prochaines années d’alternance. Il est utile de savoir pourquoi nous utilisons telle ou telle technologie. L’écosystème évolue très vite depuis le début de l’informatique, c’est pourquoi il est important de se tenir au courant des dernières technologies utilisées. Les langages informatiques ne cessent d’évoluer et de s’améliorer. Des solutions sont trouvées pour aider les développeurs à être plus rapides et efficaces. Dans cette veille technologique nous parlerons notamment du JavaScript avec React côté client et Node.js côté serveur. Ce sont les principales technologies que nous utiliserons à CGI pour le cadre du projet ReStart. Nous parlerons aussi d’autres outils nécessaires aux projets tels que Postgres pour gérer la base de données ou encore Git pour gérer les différentes versions de code source.   Javascript Histoire Brendan Eich est le fondateur de JavaScript, ancien membre de la fondation Mozilla et directeur général de Brave Software. Il a créé JavaScript en 1995 pour le compte de la Netscape Communications Corporation - qui gérait le navigateur Netscape. Peu de temps avant la sortie du langage, Netscape change le nom de LiveScript pour JavaScript. Pourquoi ? Netscape était partenaire de Sun Microsystems - qui gérait la machine virtuelle Java de plus en plus populaire. Ce nom était bénéfique pour toutes les parties. Brendan Eich s’est notamment inspiré de Java puisque la syntaxe y ressemble. En revanche, les fondamentaux des deux langages sont loin d’être similaires. Standardisation Par la suite, JavaScript a été standardisé en 1997 sous le nom de ECMAScript par ECMA International. Depuis juin 2022, nous en sommes à la 13ème édition. Aujourd’hui, JavaScript possède le plus gros écosystème grâce à son gestionnaire de dépendances npm (nous en parlerons plus tard). Fonctionnement En JavaScript, tout est objet - prototype, fonction, etc. Le langage supporte donc la programmation orienté objet (utilisant le concept de prototype) mais aussi la programmation impératif et fonctionnel. Côté client Le langage JavaScript est un langage de programmation de script principalement employé dans les pages web interactives. Il a été conçu pour être utilisé en tant que langage de script dans les navigateurs web. Les navigateurs web sont dotés d’un moteur JavaScript pour l’interpréter. Les langages informatiques HTML et CSS sont utilisés pour la structure et la mise en forme des pages web. JavaScript, quant à lui, est utilisé pour gérer le comportement de la page web en fonction de l’interaction avec l’utilisateur. Avec JavaScript, il est donc possible de créer des pages dynamiques, par opposition aux pages statiques conçues exclusivement avec du HTML et du CSS. En tant que langage à l’origine du comportement de la page web, ce langage est une partie essentielle des applications web. Tout développeur web doit maîtriser le JavaScript en plus de HMTL et CSS. Côté serveur JavaScript est aussi employé pour les serveurs web avec l’utilisation de Node.js (nous en parlerons plus tard). Sécurité Le langage JavaScript a quelque faille de sécurité. Étant donné que le JavaScript est exécuté côté client, il est possible pour une personne d’exécuter du code malveillant. Les navigateurs web en ont bien conscience et essaye de réduire ce risque avec deux restrictions. - La première est que le langage ne peut être exécuté que dans un espace à part des autres données. En effet, seul des actions relatives au web peuvent être exécuté. - Le code ne peut pas faire ce qu’il veut, à savoir accéder aux informations de types cookies des autres sites. JSON Le format JSON a été créé pour faciliter la manipulation de données avec JavaScript. JSON signifie JavaScript Object Notation. Les données sont structurées sous la forme d’objets et de liste, ce qui facilite la manipulation de données. En effet, JSON est un format de données semblable à la syntaxe des objets JavaScript. Ainsi, il existe des foncions JavaScript permettant d’extraire les données de manière très simple. Un point fort du format JSON est qu’il est utilisable indépendamment du langage JavaScript, ainsi d’autres langages de programmation peuvent tirer profit de cette syntaxe pour traiter les données ou bien même d’en générer. JSON présentant les données sous forme de chaînes de caractères, doit être converti en objet JavaScript natif. Ce n’est pas un souci puisque JavaScript fournit un objet global JSON disposant des méthodes pour assurer la conversion entre les deux.   React et MobX Introduction React est une bibliothèque JavaScript pour créer des interfaces utilisateurs. Elle a été créée par Facebook en 2013. Cette bibliothèque se base sur l’utilisation de composants réutilisables dans l’application. React permet de créer différents états d’une application facilement. Aussi, elle permet de créer différente vue pour l’application et quand les données de celle-ci changent avec l’interaction de l’utilisateur, React se charge de mettre à jour automatiquement seulement les composants concernés. Chaque composant est autonome et possède son propre état. Ainsi, il est plus facile de créer des applications complexes. La bibliothèque React est souvent confondue avec un framework. Cependant, pour lui donner ce titre, React a besoin de librairies supplémentaires. Aussi, React peut être utilisé côté serveur avec Node.js ou bien pour créer des applications mobiles avec React Native. MobX est aussi une bibliothèque JavaScript. Cette bibliothèque sera aussi importante dans le cadre de mon alternance. En effet, celle-ci est communément utilisée avec React. Cette bibliothèque permet de factoriser et de simplifier le code. Elle permet d’optimiser l’application. Couplée avec React, l’application est garantie d’être efficace. Avantages Le plus gros point fort de React est qu’il est utilisable partout. Autrement dit, il n’est pas obligé de réécrire tout le code d’une application pour intégrer React contrairement à d’autres frameworks qui sont moins souples. Le but premier de React est de permettre aux développeurs JavaScript de créer des applications complexes en un temps record. La syntaxe de React est un atout qui permet aux développeurs d’apprendre rapidement. Ainsi, milliers de développeurs à travers le monde utilisent React que ce soit en entreprise ou de manière indépendante. Avec l’utilisation de composant, il est facile de tester, débugger son application. Les composants dans l’interface graphique est une bonne manière d’envoyer et traiter les données, ce qui rend simple la gestion des potentielles erreurs. React permet l’interaction avec le DOM (Document Object Model) plus simplement en utilisant son DOM Virtuel - qui est une version simplifiée du « vrai » DOM. Cette utilisation évite donc d’interagir directement avec l’API du « vrai » DOM. Ainsi, c’est React qui gère tout puis, une fois ses instructions passées, il utilise son DOM virtuel pour le synchroniser avec le « vrai » DOM du navigateur. Cette méthode utilisée par la bibliothèque permet un temps de chargement plus faible et donc augmenter la rapidité de l’application pour l’utilisateur final. React garantit un code stable, un point crucial pour les très grands projets. Comme précisé précédemment, React utilise des composants qui communiquent entre eux avec la structure parents-enfants. Le fait que les mises à jour de composants se fassent de père en fils permet aux développeurs de réparer les bugs sans trop de difficultés puisque leurs bugs se situeront dans les composants et composants enfants concernés. Ainsi, les bugs n’auront pas (ou rarement) d’impact sur l’aspect de l’application dans son intégralité. Comparaison avec VueJS Il existe deux frameworks principaux concurrent à React. Angular et VueJS. Ces deux frameworks permettent de créer entièrement des applications web. Bien que React soit une bibliothèque, elle concurrence ces deux frameworks. Vue est un framework assez récent sortie en 2014 par un ancien employé de Google. Son objectif est d’améliorer Angular, le rendre plus simple d’utilisation et de proposer des outils personnalisés. Vue connait un grand succès, en témoignent les milliers d’étoiles sur GitHub. Certaines grandes entreprises l’utilisent comme Nintendo ou Upwork. Les développeurs ont de bon retour sur ce framework progressif. Il est dit « progressif » car le framework peut être étendu avec des packages officiels ou provenant de tiers. De plus, Vue utilise, tout comme React, le DOM virtuel pour optimiser les chargements des pages. Vue possède un point fort redoutable. Le framework permet d’intégrer un framework populaire de PHP, Laravel. Les deux utilisés ensemble permet de bien séparé le frontend et le backend. Ainsi, les équipes de développeurs peuvent ajouter leur fonctionnalité sans aucun problème. Cette architecture est donc idéale pour les projets de grande envergure avec plusieurs équipes. Conclusion En conclusion, Vue est un framework contrairement à React qui est une bibliothèque. Cependant, les deux technologies permettent de créer entièrement des applications web. On peut ajouter que React permet d’être plus souple sur le code. Ainsi, il n’est pas obligatoire d’utiliser React sur l’ensemble du projet.   Node.js Introduction Aujourd’hui, il est possible d’exécuter du JavaScript côté serveur avec Node.js. Node.js est open-source et multi-plateforme. C’est un environnement d’exécution JavaScript construit sur le moteur JavaScript V8 de Chrome. Ainsi, JavaScript est interprété en dehors du navigateur. Node.js s’exécute en un seul processus, sans créer de thread pour chaque requête. Avec node.js, il est possible de choisir la version d’ECMAScript utiliser en changeant simplement la version de node.js côté serveur. Il n’est donc pas besoin d’attendre que l’utilisateur mette à jour son navigateur puisque node.js est exécuté côté serveur. Le gestionnaire de paquet npm a fortement aidé l’écosystème node.js à proliférer avec sa structure simple. Aujourd’hui, plus d’un million de paquets open-source sont hébergés sur le registre npm que tout développeur peut utiliser librement. Node.js possède évidemment sa propre documentation. Il est cependant possible d’accéder au site nodeschool.io, un site web consacré à l’apprentissage de node.js ainsi que npm qui est installé en même temps que nodejs. L’utilisation de node.js ne se fait pas sans l’utilisation de npm. Couplés ensemble, ces outils sont très efficaces. Les exercices d’apprentissages sont de la forme d’ateliers pratiques pour l’acquisition des compétences de bases ou bien des bibliothèques populaires. Avantages Son plus gros point fort est le suivant. Les millions de développeurs front (côté client) utilisant le JavaScript dans le navigateur sont désormais capable d’écrire aussi le code côté serveur, sans à apprendre un tout autre langage (tel que PHP par exemple). Inconvénients Node.js ne devraient pas être utilisé pour effectuer des opérations lourdes côté serveur. En effet, cela est dû à son architecture non bloquante et à la gestion des évènements de node.js. Ruby on Rails (ROR) préféré à Node.js concernant les bases de données relationnelle. Il est préférable d’utiliser ROR pour les applications web soutenue par une base de données relationnelle puisque les outils de base de données relationnelle pour Node.js sont encore sous-développés par rapport à ROR. ROR étant un framework qui fournit automatiquement une configuration d’accès aux données plus mature que les outils proposés par Node.js. Cependant, avec la popularité de node.js qui continue d’augmenter, l’écart entre ces outils tend à se réduire. Node.js offre un modèle d’évènement entré-sortie à thread unique qui permet l’orchestration de tâches exécutées en parallèle. Il prend en charge plusieurs connexions sans avoir besoin d’une grande empreinte mémoire. Un peu moins utilisé, les applications de ligne de commande ou bien les API-REST. Le big data ou bien l’IoT (Internet des Objets) sont des domaines ou l’utilisation de node.js est peu utilisée, cependant leur utilisation devrait augmenter dans les années à venir. Aujourd’hui, Postgres est un système de gestion de base de données relationnel orienté objet (SGBDRO) et est le deuxième SGBD le plus utilisé à travers le monde derrière MySQL. C’est sur ce SGBDR que l’entreprise CGI a choisi de gérer sa base de données. Récemment, ils utilisaient le service de base de données propriétaire que propose Oracle. Ils sont en pleine migration. Avantages Pour commencer, PostgreSQL est open-source et gratuit, ce qui lui génère une grande communauté. Sa licence open-source permet à n’importe qui de modifier le code comme bon lui semble et de manière totalement gratuite. PostgreSQL représente l’information sous forme d’objet ce qui lui permet d’être utilisé sur les tous les principaux systèmes d’exploitation (Windows, Unix). De plus, PostgreSQL est très stable et robuste. PostgreSQL offre une véritable sémantique ACID (Atomicité Cohérence Isolation et Durabilité). Aussi, le SGBD possède son propre langage natif avec différentes fonctionnalités modernes. Il soutient aussi le type de données JSON (JavaScript Object Notation) ce qui est un bon point pour l’utilisation de JavaScript. En tant que SGBDRO, PostgreSQL permet l’héritage de table, la surcharge et permet aussi au développeur la création de type personnalisé. Inconvénients Il peut être difficile de débuter avec PostgreSQL vu la totalité de fonctionnalités proposés. La vitesse de lecture peut être assez faible à certain moment car Postgres se focalise plus sur la conformité et la compatibilité des données. Enfin, un certain nombre d’application ne sont pas compatible avec PostgreSQL. Comparaison avec Oracle Database La première comparaison évidente entre ces deux SGBD est que l’un est open-source (PostgreSQL) et l’autre est propriétaire (Oracle Database) et coûteuse. La seconde comparaison est que PostgreSQL est un système de gestion de base de données relationnel orienté objet (SGBDRO) alors qu’Oracle est un système de gestion de base de données relationnelle (SGBDR). Les deux SGBD possèdent une grande quantité de fonctionnalités mais il y a quand même avantage côté Oracle sur ce point. Côté stockage des informations, avantage Postgres. En effet, pour avoir une capacité de stockage supérieure concernant Oracle, il faut sortir le chéquier. CGI étant une entreprise gérant une quantité de données extrême, l’utilisation d’Oracle devient très coûteuse. Les deux SGBD sont utilisables sur Windows ou les systèmes Unix. Aussi, ils prennent tous les deux en charge les langages de programmation comme java, C, C++, C#. Oracle étant payant, offre un service d’aide d’urgence contrairement à PostgreSQL, qui, étant communautaire, plus d’attente est requis en cas de problème. Postgres peut être compatible avec Oracle mais la réciproque est beaucoup plus difficile. Oracle n’étant pas très compatible avec les SGBDR open-source. Oracle offre de meilleure sécurité que Postgres. En conclusion, Oracle est meilleure sur une grande partie des points, mais à condition de mettre le prix. PostgreSQL dégage cependant de nombreux avantages notamment d’être gratuit et open-source et n’a pas vraiment de chose à valoir à Oracle. Conclusion Lé décision de choisir Postgres n’a pas été simple. La migration d’une très grande quantité de données n’est pas chose facile et ce n’est qu’après longue réflexion que CGI a décidé de faire le pas en migrant sur Postgres. La principale raison pour laquelle CGI migre sur PostgreSQL est économique. En effet, la licence d’utilisation d’Oracle est très coûteuse. Plus il y a d’informations à stocker, plus la licence devient coûteuse. Avec PostgreSQL en open-source, ils font le choix comme millions de développeurs de soutenir cette communauté open-source offrant de belles performances, stable, robuste et surtout gratuite.   Git Introduction Git is a software to version your code. Versioning a project is essential. A single developer can do without versioning software, but not developers working in a team. Indeed, one person can work on a specific functionality of the project. Once the feature is finished and functional, the other people in the team can retrieve its code simply with a tool like Git. Giants like Microsoft (which has its own platform linked to Git - GitHub), IBM or even SMEs, use Git. And for good reason, it has become over time the most powerful and complete code versioning software. Avantages With Git, you can: - Revert to a previous version - See who added a change, a feature, etc. And at what time - Test a new feature without breaking the working code - And other more advanced features Git allows you to manage your code in a decentralized way. All code versions are stored on the developer's personal machine. The version history is stored in a tree of files. It is in a hidden folder at the root of the project. Each developer has the possibility to add his modification to the project by sending the source code on a remote server. The whole team has access to the remote server. With a simple git push command, the modifications made to the project are sent to the remote server. In the same way, a simple git command allows to get the last version of the source code (git pull). There is a multitude of git commands.
